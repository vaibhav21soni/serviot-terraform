# Infrastructure (Terraform)

All the AWS for this project, as Terraform modules — one VPC, two EC2 boxes (app
+ Jenkins), one RDS Postgres, security groups, and a read-only IAM user for the
reviewer. Nothing here was created by hand. This README is the how-to; the
reasoning behind the choices is in the app repo's `DEPLOYMENT.md`.

## What it builds

```
VPC ─┬─ public subnets  ── app EC2 (Nginx + both apps)  + Elastic IP
     │                     Jenkins EC2                    + Elastic IP
     └─ private subnets ── RDS Postgres (serviot_app1 + serviot_app2)
```

- **Two EC2** from one `for_each` over a map — add a box by adding a map entry.
- **Elastic IP each**, so subdomain DNS stays put across reboots.
- **RDS** private, encrypted; the app roles/databases are created by a small
  script run through the app box (RDS isn't reachable from a laptop).
- **IAM reviewer** with a hand-scoped read-only policy + a console login.

## Layout

```
main.tf            module wiring + security-group definitions
variables.tf       inputs (instances, sizes, CIDRs, backups toggle)
databases.tf       creates serviot_app1/app2 via the app box
modules/
  network/   VPC, subnets (CIDRs derived), routes
  security/  security groups (dynamic ingress from a map)
  compute/   the 2 instances, EIPs, key pair, per-role bootstrap
  database/  RDS + its security group (backups optional)
  iam/       read-only reviewer user + policy
```

## Use it

```bash
cp terraform.tfvars.example terraform.tfvars   # fill in; gitignored
export TF_VAR_master_password='...'            # keep secrets out of files
terraform init
terraform plan
terraform apply
```

Grab the reviewer credentials (send them out of band, never commit):

```bash
terraform output reviewer_username
terraform output reviewer_console_url
terraform output -raw reviewer_console_password
terraform output -raw reviewer_secret_access_key
```

## A few notes

- **Key pair** is generated by Terraform and written to `serviot-key.pem`
  (gitignored). Set `create_key_pair = false` + `existing_key_name` to reuse one.
- **`admin_ip_cidr`** locks SSH (and, if you want, Jenkins) to your IP — there's
  no default on purpose, so you can't accidentally open it to the world.
- **Backups** are off by default so teardown is one command; set
  `enable_backups = true` for production (7-day retention + final snapshot +
  deletion protection).
- **Secrets** never live in git: `terraform.tfvars`, `*.tfstate`, `.terraform/`,
  and `*.pem` are all gitignored. Pass passwords via `TF_VAR_*`.

## Not included (on purpose)

Route53 records and ACM/Let's Encrypt certs are done outside Terraform for now,
and there's no NAT gateway (the private subnets don't need outbound). Called out
so their absence is a choice, not an oversight.
